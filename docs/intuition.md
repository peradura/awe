# 직관으로 이해하는 AWE의 핵심 원리

> **한 줄 요약.** "모델이 속으로 한 걸음 더 생각하는 것(latent 스텝)"이,
> 알고 보면 "예측이 틀린 만큼 내부 기억을 고쳐 쓰는 학습 한 걸음(gradient
> descent)"과 **같은 식**이다. 그리고 그 "얼마나 틀렸나"가 바로 **surprise**다.
> AWE는 이 하나의 surprise 신호로 **얼마나 더 생각할지(깊이·halting)**와
> **기억을 얼마나 고칠지(weight 갱신)**를 동시에 조절할 수 있는지를 검증한다.

이 문서는 사전 지식이 거의 없는 사람도 AWE의 밑바탕 아이디어를 끝까지
따라올 수 있도록, **가장 밑바닥 개념부터** 차근차근 쌓아 올린다. 수식은
나오지만, 각 식마다 "말로 읽으면 무슨 뜻인지"를 항상 같이 적었다.

- **대상 독자**: 이 레포에 처음 들어온 사람, 랩 세미나 청중, 협업자.
- **읽는 순서**: 위에서 아래로. 각 절이 앞 절 위에 쌓인다.
- **더 깊이/실증 결과**: [`README.md`](../README.md), [`docs/RESULTS.md`](RESULTS.md),
  [`PROJECT.md`](../PROJECT.md), [`docs/proposal.md`](proposal.md).

## 0. 큰 그림 — 추론 시점에 "연산을 더 쓰는" 두 가지 방법

학습이 끝난 모델이 어려운 문제를 만났을 때, 정답률을 높이려고 **연산을 더
쓰는 방법**은 크게 둘이다.

1. **더 오래 생각하기 (깊이 / depth)** — 답을 내기 전에 내부 계산을 여러 번
   반복한다. 사람이 말하기 전에 속으로 여러 번 곱씹는 것과 같다.
   (예: Coconut, Geiping의 recurrent-depth)
2. **그 문제에 맞게 자신을 고치기 (가중치 / weight)** — 지금 이 입력에 대해
   내부 파라미터(기억)를 즉석에서 조금 갱신한다. 이걸 **Test-Time Training
   (TTT)** 이라 부른다. (예: Titans, PonderTTT)

기존 연구는 보통 **둘 중 하나만** 구동한다. AWE의 질문은:

> **이 둘을 "놀란 정도(surprise)"라는 신호 하나로 동시에 굴릴 수 있는가?**
>
> - surprise ↑ (아직 이해 못 함) → **더 생각하고** + 기억을 **더 고친다**
> - surprise ↓ (수렴함)          → **멈추고**   + 기억 갱신도 **멈춘다**

이 문서의 나머지는 "왜 이 두 개가 원래 하나의 뿌리에서 나오는가"를 설명한다.
그 뿌리가 바로 **"latent 스텝 = gradient descent 한 스텝"** 이라는 등식이다.

## 1. "latent"이 뭔가 — 모델의 머릿속 표현

딥러닝에서 **latent(잠재)** = **겉으로 안 보이는, 모델 내부의 숫자 표현**이다.
거의 **hidden(숨은)** 과 같은 말.

| 구분 | 무엇 | 사람이 읽을 수 있나 |
|---|---|---|
| **겉 (관측)** | 입력 단어, 출력 단어 | O — 사람이 읽음 |
| **속 (latent)** | 그 사이에서 계산되는 숫자 벡터들 | X — 모델만 아는 내부 언어 |

**비유.** 번역가가 "apple"을 "사과"로 옮길 때:
- 입력 "apple", 출력 "사과" = **겉**
- 머릿속에 순간 떠오르는 "둥글고 빨간 먹는 것" 같은 개념 덩어리 = **latent**

이 머릿속 개념은 영어도 한국어도 아닌 번역가만의 내부 표현이다. 그게 latent다.

### 1-1. 그래서 "latent 스텝"은?

**답을 내기 전에, 이 내부 표현(벡터)을 한 번 더 굴려 생각하는 한 단계.**

```
입력 → [latent 스텝] → [latent 스텝] → ... → 출력
         (속으로 생각)     (또 생각)
```

- 사람으로 치면 **말하기 전에 속으로 한 번 더 곱씹는 것**.
- 이 곱씹기를 **몇 번 할지**(= 깊이/depth), **각 곱씹기가 정확히 무슨 계산을
  하는지** — 이 두 가지가 아래에서 "기억 고쳐쓰기"와 연결된다.

## 2. 모델 안의 "작은 학생" — 연상 메모리 셋업

모델 안에 **공책 한 권을 든 작은 학생**이 산다고 상상하자. 이 공책이 곧
**메모리** $W$ (하나의 행렬)다. 학생이 하는 일:

- 토큰(단어)이 하나씩 들어온다 = **문제가 하나씩 주어진다**.
- 각 토큰은 두 부분으로 쪼갤 수 있다:
  - **key** $k$ — "이건 무엇에 대한 질문인가" (열쇠/단서)
  - **value** $v$ — "그 질문의 정답은 무엇인가" (내용)
- 학생은 공책 $W$를 보고 **key로 정답을 떠올려 본다**: 예측 = $Wk$.
- 그리고 **진짜 정답 $v$와 비교**한다.

이게 **연상 메모리(associative memory)** 다: 열쇠 $k$를 주면 그에 얽힌 내용
$v$를 떠올리는 저장소. 그래서:

$$
\underbrace{Wk}_{\text{내 예측}} \quad\text{vs}\quad \underbrace{v}_{\text{정답}}
$$

- $Wk$ 가 **"예측"** 인 이유: 공책 $W$로 열쇠 $k$를 조회해 떠올린 결과라서.
- $v$ 가 **"정답"** 인 이유: 그 토큰이 실제로 담고 있던 내용이라서.

둘의 차이 $(Wk - v)$ 가 **예측 오차**다. 학생이 얼마나 틀렸는가.

---

## 3. "틀린 만큼 고쳐 쓰기" = gradient descent

학생이 공책을 얼마나 못 쓰고 있는지를 숫자 하나로 재는 게 **손실(loss)**:

$$
\mathcal{L}(W) = \tfrac{1}{2}\,\lVert Wk - v \rVert^2
\qquad(\text{"정답을 얼마나 못 떠올리나"})
$$

이 손실을 줄이는 표준 방법이 **gradient descent(경사하강)** — 딥러닝 학습의
바로 그 방법이다. 손실의 기울기(gradient)를 구해 그 반대 방향으로 조금 간다.

$$
\nabla_W \mathcal{L} = (Wk - v)\,k^\top
\qquad\Longrightarrow\qquad
W \;\leftarrow\; W - \eta\,\underbrace{(Wk - v)\,k^\top}_{\text{gradient}}
$$

**말로 읽으면:** (내 예측 $Wk$) 빼기 (정답 $v$) = 오차, 그 오차만큼 공책 $W$를
$\eta$(학습률)의 세기로 고쳐 쓴다. **틀린 만큼 반성해서 기억을 수정**하는 것.

> 핵심: 이건 원래 **훈련 때** 하는 일이다. 그런데 아래에서 보듯, 어떤 모델
> 구조는 **실행(추론) 중에도 매 토큰마다** 이 고쳐쓰기를 한다. 그게 TTT다.

## 4. 핵심 등식 — latent 스텝이 "그" gradient descent와 같은 식이다

이제 이 문서의 심장이다. **모델이 기억을 굴리는 원래 공식**이, 위의 학습
공식과 **글자 그대로 같다**는 것을 보인다.

linear attention / fast-weight 계열 모델이 메모리를 갱신하는 공식은
**delta rule(델타 규칙)** 이라 불리며 이렇게 생겼다:

$$
W_{\text{new}} = W + \beta\,(v - Wk)\,k^\top
$$

이걸 3절의 gradient descent 스텝과 나란히 놓아 보자:

$$
\begin{aligned}
\text{학습(GD):}\quad & W - \eta\,(Wk - v)\,k^\top \\[2pt]
\text{모델(delta rule):}\quad & W + \beta\,(v - Wk)\,k^\top
\end{aligned}
$$

$(v - Wk) = -(Wk - v)$ 이므로 부호를 정리하면:

$$
W + \beta\,(v - Wk)\,k^\top \;=\; W - \beta\,(Wk - v)\,k^\top
$$

→ **두 식이 완전히 같다.** 모델의 학습률 $\beta$ 가 GD의 학습률 $\eta$ 역할.

> **이게 "latent 스텝이 gradient descent를 근사한다"의 정체다.**
> 구조 설계자는 그냥 "기억을 업데이트하는 방법"이라고 만들었는데, 수식을
> 뜯어보니 **예측 오차를 줄이는 gradient descent 한 스텝과 똑같았다.**

### 4-1. 왜 신기한가 (직관)

```
겉모습:  그냥 벡터를 곱하고 더하는 계산 (forward pass)
실체:    "예측 → 오차 확인 → 그만큼 기억 수정" = 학습
```

모델은 **역전파(backprop)를 돌리는 게 아닌데도**, 순전파(forward) 계산만으로
결과적으로 gradient descent와 똑같은 일을 하고 있다. 그래서 이 계열 연구를
**"모델이 몰래 학습한다(secretly learn)"** 고 부른다.

### 4-2. 왜 "근사(approximate)"라고 하나 — 완전 일치가 아닌 경우

- **delta rule**은 위처럼 **정확히** gradient descent 1스텝이다 (등호 성립).
- 하지만 실제 Transformer 층의 **softmax attention**은 이 계산을 **비슷하게
  흉내 낼 뿐** 딱 떨어지진 않는다. 그래서 일반적으로는 "근사한다"고 말한다.
- von Oswald 등(2023)은 **선형 self-attention 한 층의 forward pass가 회귀
  문제에 대한 gradient descent 한 스텝을 (근사적으로) 구현**함을 보였다.
  즉 "생각하는 한 걸음"이 "학습하는 한 걸음"으로 해석된다는 걸 실제 어텐션
  구조에서 확인한 것.

**정리:** latent 스텝을 여러 번 밟는 것 ≈ gradient descent를 여러 스텝 밟아
내부 기억을 그 입력에 맞게 점점 최적화하는 것. **더 생각할수록 기억이 더
잘 맞춰진다.**

## 5. surprise의 정체 — gradient의 크기, 그 자체

여기서 흔히 하는 오해를 바로잡자.

- ❌ **오해**: "surprise가 있어야 → gradient descent가 일어난다" (surprise가 방아쇠)
- ✅ **실제**: "gradient descent 식 안에 들어있는 **오차항 자체가 곧 surprise**다"

둘은 **원인–결과가 아니라, 같은 것을 두 이름으로 부르는 것**이다.

$$
W_{\text{new}} = W - \eta\,\underbrace{(Wk - v)\,k^\top}_{\text{gradient} \;=\; \text{surprise}}
$$

- 이 gradient 항 = 손실이 얼마나 가파른가 = **모델이 얼마나 놀랐나(surprise)**.
- Titans(2501.00663) 논문은 실제로 이 gradient를 **surprise라고 정의**한다.
- 그러니 "surprise가 GD를 일으킨다"가 아니라 →
  **gradient descent 스텝의 크기 = surprise의 크기**.

```
많이 틀림   → gradient 큼   → surprise 큼   → 기억을 크게 갱신
잘 맞음     → gradient 작음 → surprise 작음 → 거의 안 갱신
```

### 5-1. 자동 스케일링 — 스위치가 필요 없다

surprise가 0에 가까우면 gradient가 0에 가까우므로 **갱신이 저절로 미미해진다.**
별도의 on/off 스위치가 필요 없다. **놀람의 크기가 곧 갱신의 세기**라서, 익숙한
입력엔 자동으로 손을 덜 대고, 낯선 입력엔 자동으로 크게 반응한다.

## 6. AWE로의 연결 — 하나의 신호, 두 개의 손잡이

지금까지의 그림을 합치면 AWE의 아이디어가 자연스럽게 나온다.

surprise는 **이미 gradient 안에 존재하는 양**이다 — 따로 만들 필요 없이
**공짜로 얻는다.** 그렇다면 그 **하나의 양**으로 두 가지를 동시에 정할 수 있다:

| 질문 | 무엇으로 답하나 | 대응하는 손잡이 |
|---|---|---|
| 기억을 **얼마나 세게** 고칠까? | 놀란 만큼 | **(W) Weight** — TTT 갱신 |
| **몇 번 더** 생각(latent 스텝)할까? | 놀랐으면 더, 익숙하면 그만 | **(E) Exit** — 깊이/halting |

- **(W) Weight 갱신**은 5절에서 봤듯 **이미 gradient descent가 자동으로** 한다.
- **(E) Exit / halting**에 **같은 surprise 신호를 재사용**하자는 것이 AWE의
  새로운 제안이다. 즉 새 신호를 발명하는 게 아니라, **gradient descent가
  이미 뱉고 있는 surprise를 "언제 그만 생각할지"에도 꽂는 것.**

이래서 이름이 **AWE = Adaptive Weight & Exit** — *"one surprise signal,
two test-time knobs"*.

```
          ┌───────────────────────────────┐
 latent   │   surprise = ‖gradient‖        │
 스텝  ──▶ │   (연상 메모리 재구성 오차)      │
          └───────────────┬───────────────┘
                          │  하나의 신호가
              ┌───────────┴───────────┐
              ▼                       ▼
   (W) 기억을 얼마나 갱신?      (E) 몇 스텝 더 생각? / 언제 멈춤?
   = TTT weight update        = 깊이 halting
```

## 7. 실험은 어떻게 설계했나 (Experiment Design)

지금까지(§1–6)는 **"왜 이 아이디어가 그럴듯한가"** 라는 이론적 동기였다.
아름답다고 참인 건 아니다. AWE는 이걸 **작은 합성 과제 위에서 실제로 검증**한다.
설계의 핵심 전략은 **"사다리"** — 손잡이를 하나씩 격리해 각각 되는지 확인한
뒤, 합치고, 마지막에 외부 과제로 전이되는지 본다.

### 7-1. 검증 사다리 — 무엇을 왜 각각 보나

| 단계 | 실험 | 격리한 것 | 답하려는 질문 |
|---|---|---|---|
| **Part 1** | in-context 도달성(reachability) | **깊이(E)만** | 더 깊이 생각하면 더 어려운 문제를 푸나? halting이 난이도를 따라가나? |
| **Part 2** | 숨은 규칙(hidden-rule) | **가중치(W)만** | 추론 중 기억을 쌓으면 실제로 정답률이 오르나? |
| **Part 3** | 부분관측 도달성(joint reachp) | **둘 다** | 한 신호로 깊이+가중치를 **동시에** 굴리면 잘 되나? |
| **Part 4** | halting 신호 **bake-off** | 신호 비교 | Part 3이 실패했다면, **어떤 신호**가 halting에 맞나? |
| **Part 5** | MQAR(외부 표준 과제) | 전이 | Part 4의 결론이 **다른 과제로도** 전이되나? |

**과제 설명 (직관):**
- **도달성(reachability)** — 함수 그래프에서 "여기서 K번 따라가면 어디 도착?".
  **K(hop 수) = 난이도**. K가 크면 더 많은 latent 스텝이 필요 → 깊이 손잡이를
  자연스럽게 시험함.
- **숨은 규칙(hidden-rule)** — 감춰진 순열 π를, 질문 스트림에 **조금씩만** 흘림.
  각 질문은 **이전 질문들에서 쌓인 기억으로만** 풀 수 있음 → 기억(가중치)
  손잡이만 시험함.
- **부분관측 도달성(reachp / reachp2)** — 위 둘을 합침: 그래프를 부분만 보여주고
  기억에 누적시키며, hop 수 K로 깊이도 요구. `reachp2`는 base 모델을 강화한 버전
  (K-커리큘럼 + 보조손실 + 폭 d=256).
- **MQAR** — associative recall의 **학계 표준 벤치마크**. 우리 합성 과제 밖에서도
  결론이 서는지 보는 외부 시험대(single-hop / multi-hop).

### 7-2. 비교 구성 (4-way ablation)

각 과제에서 네 가지를 같은 조건으로 맞대결시킨다:

| 구성 | 깊이 halting | 기억 지속 | 뜻 |
|---|---|---|---|
| **fixed** | ✗ (고정) | ✗ (리셋) | 아무것도 안 함 (바닥선) |
| **+halt** | ✓ | ✗ | 깊이 손잡이만 |
| **persist** | ✗ | ✓ | 기억 손잡이만 |
| **both (AWE)** | ✓ | ✓ | 둘 다 (우리 제안) |

### 7-3. "언제 멈출지"를 정하는 후보 신호들

halting = "latent 스텝을 언제 그만 밟을지". 그 **판단 재료(관측량)** 로 여러
후보를 두고 공정하게 경쟁시켰다 (Part 4의 핵심):

| 신호 | 뜻 (말로) | 이론상 정체 |
|---|---|---|
| `entropy` | 답 분포가 얼마나 헷갈리나 (기존 관행) | readout 확신도 |
| `recon` | 재구성 오차 = **메모리가 얼마나 못 맞췄나** | **§5의 그 surprise** (문자 그대로의 명제) |
| `dstate` | latent 상태가 스텝 사이 **얼마나 변했나** ‖Δs‖² | ≈ ‖∇ₛL‖ = **surprise gradient의 크기** |
| `conv` | 예측(readout)이 스텝 사이 **얼마나 안 변하나** (sym-KL) | **수렴** 신호 (Geiping식) |
| `rnorm`, `dent` | 보조 후보 (읽기 노름 / entropy 변화량) | — |

> 여기서 이 문서의 핵심과 직접 연결된다: **`recon`이 바로 §5에서 말한 "재구성
> 오차 = surprise"** 이고, **`dstate`가 §4–5의 "gradient 크기 ‖∇ₛL‖"** 에
> 해당한다. 즉 우리의 이론 이야기가 예측하는 halting 신호는 `recon`/`dstate`다.
> 과연 그게 이기는지가 실험의 진짜 쟁점.

### 7-4. 공정성 규율 (숫자를 믿게 만드는 장치)

- **10 seeds** — 모든 결과는 서로 다른 난수 10회의 평균±표준편차.
- **held-out tau** — "얼마나 낮아야 멈춤"의 임계값 τ를 **평가와 다른 별도 배치**
  에서 보정 (train 데이터로 τ를 정하고 test에서 유리하게 고르는 부정 방지).
- **shuffle null (헛것 대조군)** — 신호를 블록 안에서 뒤섞어도 같은 성능이면 그건
  진짜 "예제별 정보"가 아님. 이걸 이겨야 신호가 유의미.
- **tau 선택 목적 = fewest-steps-within-slack** — "정확도 손해 ≤1pp 안에서 **가장
  적은 스텝**"을 고름 (초기 버전은 argmax-accuracy를 썼는데, 이게 Part 4a의
  bimodal 잡음을 만든 원인 → Part 4c에서 교정).

## 8. 실험 결과 (Results)

**전체 스토리는 "실패 → 진단 → 반전 → 전이"의 3막극이다.** 순서대로 본다.
(모든 수치는 10 seeds 평균; 원본: [`docs/RESULTS.md`](RESULTS.md).)

### Part 1 — 깊이만: 잘 된다 ✅

더 깊이 생각할수록 정확도가 오르고(r=1: **15.7%** → r=12: **100%**), 멈추는
시점이 난이도 K를 거의 완벽히 따라간다:

$$\text{corr}(K,\ \text{halt step}) = +0.997 \pm 0.001$$

→ 과제가 "깊이 손잡이"를 시험하기에 올바른 난이도 구조를 가짐을 확인.
(단, 여기 halting은 사후 수렴 판정이라 surprise 신호를 직접 시험한 건 아직 아님.)

### Part 2 — 기억만: 가장 강한 결과 ✅

기억을 **지속(persist)** 하면 스트림을 따라 정답률이 **5% → 81%** 로 오른다.
리셋하면 계속 찍기 수준(~5%)에 머묾 → 기억이 **필수**임이 명확.

| 구성 | 정확도 | 평균 latent 스텝 |
|---|---|---|
| fixed / +halt (리셋) | ~5% | — |
| **persist** | 50.1% | 8.0 |
| **both** | 50.0% | **2.4** |

→ `both`는 **같은 정확도를 8.0 대신 2.4 스텝**으로 (≈3.3× 적은 조회). 스트림을
따라 스텝이 **7.95 → 1.21** 로 줄어드는 amortization도 확인. 또 `corr(정답가능,
entropy) = −0.96` — 답이 기억에 있을 때 정확히 확신함(잘 보정된 신호).

### Part 3 — 둘 다 합치니: 실패 🟡 (그러나 원인이 있다)

여기서 문제가 터진다. **halting을 켜면 정확도가 깎인다:**

| 구성 | 정확도 | 평균 스텝 |
|---|---|---|
| fixed | 21.0% | 10.0 |
| +halt | 16.0% | 3.2 |
| **persist** | **35.3%** | 10.0 |
| **both (AWE)** | 25.7% | 5.4 |

- `both`(25.7%)가 `persist`(35.3%)보다 **−9.6pp**. 일찍 멈추긴 하는데(5.4<10),
  **확신에 차서 틀린 채로(confident-but-wrong) 조기 종료**하는 예제가 많았음.
- 당시 결론: "한 신호로 둘 다"의 **순진한 버전은 깨진다.** 하지만 이게
  **메커니즘의 실패인지, 신호 선택의 실패인지**가 다음 질문으로 남음.

### Part 4 — 진단: "신호 선택" 문제였다 ✅ (반전)

base 모델을 강화(reachp2: 커리큘럼+보조손실+d=256)해 천장을 **72.0%** 로 올린 뒤,
**같은 궤적 위에서 여러 halting 신호를 맞대결**시켰다.

**Part 4a (강한 학습기, 10 seeds) — 핵심 표:**

| halt 신호 | 정확도 | 평균 스텝 | persist 대비 | 조기-오답 |
|---|---|---|---|---|
| `dstate` (‖Δs‖²) | **72.0%** | 5.93 | **−0.0pp** | **1.1%** |
| `conv` (sym-KL) | 71.9% | 5.00 | **−0.0pp** | 3.6% |
| `entropy` (기존) | 66.4% | 5.44 | −5.6pp | 8.5% |
| `recon` (재구성 오차) | 66.3% | 5.39 | −5.6pp | 8.8% |

→ **−5.6pp의 손해는 신호 특유의 문제.** `entropy`·`recon`은 공격적으로 멈추되
**잘못 보정**돼 ~9%가 "확신하고 틀린 채" 조기 종료. 반면 **수렴 계열
(`conv`, `dstate`)은 정확도를 천장 그대로 유지**.

**Part 4c (정본/canonical, 10 seeds) — 엄격한 프로토콜로 통일:**

| 신호 | 정확도 | 스텝 | slack 내 작동점 존재? |
|---|---|---|---|
| **`conv`** | **71.2%** | **3.89±0.09** | ✅ **10/10** |
| `dstate` | 71.0% | 5.12 | ✅ 10/10 |
| `dent` | 68.8% | 5.70 | ❌ 0/10 |
| `rnorm` | 68.2% | 5.57 | ❌ 0/10 |
| `recon` | 67.1% | 5.45 | ❌ 0/10 |
| `entropy` | 66.2% | 5.43 | ❌ 0/10 |

- **`conv`는 ~35% 연산을 절약**(3.89/6 스텝)하면서 정확도 손해 ≤1pp, **10/10
  seed 균일**. Part 4a에서 보였던 bimodal/보수성은 **τ 규칙의 잡음**이었고 여기서
  해소됨.
- **구별이 이분법적**이 됨: slack(≤1pp) 안에서 작동점을 찾는 신호는 **수렴 계열
  (`conv`/`dstate`)뿐** (10/10). 나머지 넷은 0/10.
- `conv`는 shuffle null(헛것 대조군)을 **+7.4pp** 이김 → 진짜 예제별 정보 담음.

### Part 5 — 외부 전이: 결론이 옮겨간다 ✅

우리 합성 과제 밖, 학계 표준 **MQAR** 로도 같은 결론이 재현되는가?

- **Multi-hop MQAR** (hop 수 = 난이도): **수렴 계열이 entropy/recon을 +2.5pp,
  10/10 seed로 이김** (2.5 vs 5.4 스텝, 절반 이하 연산). 깊이가 hop 수 따라 증가
  (2.75→3.51). → reachp의 결론이 **외부 과제로 일반화**됨.
- **Single-hop MQAR**: `conv`가 천장을 **2.53/6 스텝(~58% 절약)** 으로 전이하지만,
  이 과제는 신호를 **구별하지 못함**(오답이 애초에 못 푸는 문제라 조기 종료가
  무해). → 전이는 되나, 구별력은 multi-hop에서만 드러남.

## 9. 솔직한 종합 판정 (Honest Verdict)

3막극의 결론을 정직하게 정리하면:

- ✅ **깊이(E) 손잡이는 성립** — 단, 올바른 신호는 **수렴(`conv`)**. "더 생각할지"는
  *예측이 더 안 변하는가*로 판단해야 잘 된다.
- ✅ **가중치(W) 손잡이는 성립** — 재구성 오차로 게이팅한 delta-rule 갱신이 잘 작동
  (초기 발산은 정규화·decay·작은 inner-lr로 안정화).
- ❌ **강한 명제는 죽었다** — "**하나의 재구성 오차 스칼라(`recon`)가 두 손잡이를
  모두 최적 구동**"은 성립 안 함. `recon`은 모든 joint 과제에서 halting에 **패배**.
  연산을 아끼는 halter(`conv`)는 **읽기(readout) 통계**라 write 역할이 없음.

> **핵심 교훈:** **깊이와 가중치는 서로 다른 관측량을 원한다.**
> 깊이 ← 수렴(`conv`), 가중치 ← 재구성-미스(`recon`).

그럼 §4–5에서 쌓은 **"latent 스텝 = GD, surprise = gradient"** 이야기는 어떻게
됐나? — **부분적으로 살아있다.** 그 이론이 가리키는 신호 `dstate`(≈‖∇ₛL‖)는
정확도를 **보존**한다(−0.0pp). 다만 *가장 연산 효율적인* halter는 아니었다
(보수적으로 멈춤). 즉 **"두 손잡이 = 하나의 메모리 손실의 두 gradient
(∇ₛL / ∇_WL)"** 라는 약한 해석은 **모순은 아니지만 아직 미검증**이다 —
이게 "발견"이 되려면 **halt 신호가 write 크기를 실제로 예측함**을 보여야 한다
(다음 실험, [`docs/mqar_design.md`](mqar_design.md)의 probe).

**꼭 붙일 단서 (over-claim 방지):**
- "연산"은 **latent 조회 스텝만** 셈 (delta-rule write FLOPs는 halting과 무관).
- 모델은 **0.2–0.9M 파라미터, 합성 과제** — 이건 **메커니즘 파일럿**이지 LLM
  규모 증거가 아니다.

**다음 방향:** 지금의 깊이-신호 진단 자체가 이미 논문 한 편의 단위다 —
*"Convergence, not surprise, is the right halting observable for TTT-style memory
reasoners"*. 통합(unification) 프로그램은 write-크기 예측 probe가 성공해야만
되살아난다.

## 10. 용어 미니 사전

| 용어 | 뜻 (한 줄) |
|---|---|
| **latent / hidden** | 입력·출력 사이, 모델 내부의 숫자 벡터 표현 (사람이 못 읽음) |
| **latent 스텝** | 답을 내기 전 내부 표현을 한 번 더 굴려 생각하는 한 단계 |
| **연상 메모리** | 열쇠 $k$를 주면 얽힌 내용 $v$를 떠올리는 저장소 $W$ |
| **key $k$ / value $v$** | 토큰의 "질문" 부분 / "정답" 부분 |
| **손실(loss) $\mathcal{L}$** | 예측이 정답에서 얼마나 벗어났나를 재는 숫자 |
| **gradient descent** | 손실의 기울기 반대로 조금 가서 손실을 줄이는 학습 스텝 |
| **delta rule** | fast-weight 메모리 갱신 공식; GD 1스텝과 동일 |
| **surprise** | 그 gradient의 크기 = 모델이 얼마나 놀랐나 (= 얼마나 틀렸나) |
| **TTT (Test-Time Training)** | 추론 중에 입력에 맞춰 가중치를 즉석 갱신 |
| **halting / Exit** | latent 스텝을 언제 그만 밟을지 결정 (= 깊이 조절) |
| **ablation** | 손잡이를 켜고/끄며(fixed·+halt·persist·both) 기여를 분리하는 비교 |
| **seed** | 난수 시드. 10 seeds = 서로 다른 초기화 10회로 평균±표준편차 |
| **tau (τ)** | halting 임계값. 신호가 이보다 낮아지면 멈춤. held-out에서 보정 |
| **`conv`** | 예측이 스텝 사이 안 변하는 정도(수렴). **깊이의 올바른 신호로 판명** |
| **`dstate`** | latent 상태 변화량 ‖Δs‖² ≈ ‖∇ₛL‖. 이론이 가리키는 surprise gradient |
| **`recon`** | 재구성 오차 = 문자 그대로의 surprise. write엔 맞지만 halting엔 패배 |
| **pp** | percentage point. 정확도 차이 단위 (예: −5.6pp) |

## 11. 참고 문헌 (읽을거리)

**"forward 계산이 곧 학습" 계보**
- Schlag et al. 2021, *Linear Transformers Are Secretly Fast Weight Programmers* — delta rule = fast weight = 숨은 gradient descent.
- von Oswald et al. 2023, *Transformers Learn In-Context by Gradient Descent* — 어텐션 한 층의 forward가 회귀 GD 한 스텝을 근사.
- Sun et al. 2024, *TTT layers* (2407.04620) — hidden state 자체가 test-time에 self-supervised loss로 학습되는 모델. (원조: Sun et al. 2020, 1909.13231)

**surprise / 메모리 갱신**
- Behrouz et al. 2024, *Titans* (2501.00663) — gradient를 surprise로 정의, momentum(과거 surprise 누적)과 forgetting 포함.
- Sim 2026, *PonderTTT* (2601.00894) — TTT 재구성 손실로 weight 갱신을 이진 게이팅.

**latent 깊이 / halting**
- Meta 2024, *Coconut* (2412.06769) — latent 추론(스텝 수 고정).
- Geiping et al. 2025, *recurrent-depth* (2502.05171) — latent 깊이 스케일링, per-token 종료를 KL-수렴으로.
- *FR-Ponder* (2509.24238) — 적응형 latent 깊이 halting을 RL(GRPO)로.
